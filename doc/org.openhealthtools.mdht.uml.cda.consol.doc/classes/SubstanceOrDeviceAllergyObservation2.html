Substance Or Device Allergy Observation2

[Observation: templateId 2.16.840.1.113883.10.20.24.3.90]

This template reflects a discrete observation about a patient's allergy or intolerance to a substance or device. Because it is a discrete observation, it will have a statusCode of "completed". The effectiveTime, also referred to as the "biologically relevant time" is the time at which the observation holds for the patient. For a provider seeing a patient in the clinic today, observing a history of penicillin allergy that developed five years ago, the effectiveTime is five years ago. The effectiveTime of the Substance or Device Allergy - Intolerance Observation is the definitive indication of whether or not the underlying allergy/intolerance is resolved. If known to be resolved, then an effectiveTime/high would be present. If the date of resolution is not known, then effectiveTime/high will be present with a nullFlavor of "UNK".

  1. SHALL contain exactly one [1..1] templateId ( CONF:1098-16305, CONF:1098-16306, CONF:1098-32527 ) such that it
    1. SHALL contain exactly one [1..1] @root="2.16.840.1.113883.10.20.24.3.90"
    2. SHALL contain exactly one [1..1] @extension="2014-06-09"
  2. SHALL conform to Substance Or Device Allergy Observation template (templateId: 2.16.840.1.113883.10.20.24.3.90)
  3. SHALL contain exactly one [1..1] effectiveTime (CONF:1098-16309)

    Note: The effectiveTime/low (a.k.a. "onset date") asserts when the allergy/intolerance became biologically active. The effectiveTime/high (a.k.a. "resolution date") asserts when the allergy/intolerance became biologically resolved.


    If the allergy/intolerance is known to be resolved, but the date of resolution is not known, then the high element SHALL be present, and the nullFlavor attribute SHALL be set to 'UNK'.

  4. SHALL contain exactly one [1..1] value with @xsi:type="CD" (CONF:1098-16312), which SHALL be selected from ValueSet Allergy/Adverse Event Type 2.16.840.1.113883.3.88.12.3221.6.2 DYNAMIC (CONF:1098-16317)

    Note: Many systems will simply assign a fixed value here (e.g., "allergy to substance").

  5. SHALL contain exactly one [1..1] code (CONF:1098-16345)/@code="ASSERTION" Assertion (CodeSystem: 2.16.840.1.113883.5.4 HL7ActCode) (CONF:1098-16346, CONF:1098-32171)
  6. SHOULD contain zero or more [0..*] participant such that it (CONF:1098-16318)
    1. SHALL contain exactly one [1..1] @typeCode="CSM" (CONF:16319, R2.0=CONF:1098-16319)
    2. SHALL contain exactly one [1..1] participantRole such that it (CONF:1098-16320)
      1. SHALL contain exactly one [1..1] @classCode="MANU" (CONF:16321, R2.0=CONF:1098-16321)
      2. SHALL contain exactly one [1..1] playingEntity such that it (CONF:1098-16322)
        1. SHALL contain exactly one [1..1] @classCode="MMAT" (CONF:16323, R2.0=CONF:1098-16323)
        2. SHALL contain exactly one [1..1] code (CONF:1098-16324)
        3. MAY satisfy: This constraint is being overridden as it has been removed in v2 and should never fire
        4. MAY satisfy: This constraint is being overridden as it has been removed in v2 and should never fire
        5. MAY satisfy: This constraint is being overridden as it has been removed in v2 and should never fire
        6. MAY satisfy: This constraint is being overridden as it has been removed in v2 and should never fire
        7. MAY satisfy: This constraint is being overridden as it has been removed in v2 and should never fire
        8. MAY satisfy: This constraint is being overridden as it has been removed in v2 and should never fire
  7. SHOULD contain [0..*] entryRelationship such that it (CONF:1098-16337, CONF:1098-16339, CONF:1098-16340)
    1. Contains @typeCode="MFST" MFST
    2. Conforms to Reaction Observation2 (templateId: 2.16.840.1.113883.10.20.22.4.9:2014-06-09)
  8. SHOULD contain [0..1] entryRelationship such that it (CONF:1098-16341, CONF:1098-16342, CONF:1098-16344, DSTU:710)
    1. Contains @typeCode="SUBJ" SUBJ
    2. Conforms to Severity Observation2 (templateId: 2.16.840.1.113883.10.20.22.4.8:2014-06-09)
  9. SHOULD contain [0..*] author such that it (CONF:1098-31144)
    1. Conforms to Author Participation (templateId: 2.16.840.1.113883.10.20.22.4.119)
  10. SHOULD contain [0..1] entryRelationship such that it (CONF:1098-NEW, DSTU:710)
    1. Contains @typeCode="SUBJ" SUBJ
    2. Conforms to Criticality Observation (templateId: 2.16.840.1.113883.10.20.22.4.145)
  11. entryRelationship with target entry Reaction Observation (V2) SHALL contain exactly one [1..1] @inversionInd="true" True (CONF:1098-16338)
  12. entryRelationship with target entry Severity Observation (V2) SHALL contain exactly one [1..1] @inversionInd="true" True (CONF:1098-16343, DSTU:710)
  13. entryRelationship with target entry Criticality Observation SHALL contain exactly one [1..1] @inversionInd="true" True (CONF:NEW, DSTU:710)
