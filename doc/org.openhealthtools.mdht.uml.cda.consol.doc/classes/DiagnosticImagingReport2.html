Diagnostic Imaging Report (V3)

[ClinicalDocument: templateId 2.16.840.1.113883.10.20.22.1.5]

A Diagnostic Imaging Report (DIR) is a document that contains a consulting specialist's interpretation of image data. It conveys the interpretation to the referring (ordering) physician and becomes part of the patient's medical record. It is for use in Radiology, Endoscopy, Cardiology, and other imaging specialties.

Diagnostic Imaging Report (V3) Header Constraints

Diagnostic Imaging Report (V3) Body Constraints

  1. SHALL contain exactly one [1..1] templateId ( CONF:1198-8404, CONF:1198-10042, CONF:1198-32515 ) such that it
    1. SHALL contain exactly one [1..1] @root="2.16.840.1.113883.10.20.22.1.5"
    2. SHALL contain exactly one [1..1] @extension="2015-08-01"
  2. SHALL conform to US Realm Header2 template (templateId: 2.16.840.1.113883.10.20.22.1.1:2015-08-01)
  3. SHALL contain exactly one [1..1] id (CONF:1198-30932)
  4. SHALL contain exactly one [1..1] code (CONF:1198-14833), which SHOULD be selected from ValueSet LOINC Imaging Document Codes 1.3.6.1.4.1.12009.10.2.5 DYNAMIC (CONF:1198-14834)

    Preferred code is 18748-4 LOINC Diagnostic Imaging Report

  5. SHALL contain [0..0] informant such that it (CONF:1198-8410)
    1. Conforms to Informant12 from CDA
  6. MAY contain [0..*] informationRecipient such that it (CONF:1198-8411)

    informationRecipient

    1. Conforms to Information Recipient from CDA
  7. MAY contain zero or one [0..1] participant such that it (CONF:1198-8414)
    1. SHALL contain exactly one [1..1] associatedEntity such that it (CONF:1198-31198)
      1. SHALL contain exactly one [1..1] associatedPerson such that it (CONF:1198-31199)
        1. SHALL contain exactly one [1..1] name (CONF:1198-31200)
  8. MAY contain zero or more [0..*] inFulfillmentOf such that it (CONF:1198-30936)
    1. SHALL contain exactly one [1..1] order such that it (CONF:1198-30937)
      1. SHALL contain at least one [1..*] id (CONF:1198-30938)

        Note: DICOM Accession Number in the DICOM imaging and report data

  9. SHALL contain exactly one [1..1] documentationOf such that it (CONF:1198-8416)
    1. SHALL contain exactly one [1..1] serviceEvent such that it (CONF:1198-8431)
      1. SHALL contain exactly one [1..1] @classCode="ACT" Act (CodeSystem: 2.16.840.1.113883.5.6 HL7ActClass) (CONF:1198-8430)
      2. SHOULD contain zero or more [0..*] id (CONF:1198-8418)
      3. SHALL contain exactly one [1..1] code (CONF:1198-8419)
      4. SHOULD contain zero or more [0..*] performer, where its type is Physician Reading Study Performer2 (CONF:1198-8422)
      5. The value of serviceEvent/code SHALL NOT conflict with the ClininicalDocument/code. When transforming from DICOM SR documents that do not contain a procedure code, an appropriate nullFlavor SHALL be used on serviceEvent/code (CONF:1198-8420)
  10. MAY contain zero or one [0..1] relatedDocument such that it (CONF:1198-8432)
    1. SHALL contain exactly one [1..1] parentDocument such that it (CONF:1198-32089)
      1. SHALL contain exactly one [1..1] id (CONF:1198-32090)
    2. When a Diagnostic Imaging Report has been transformed from a DICOM SR document, relatedDocument/@typeCode SHALL be XFRM, and relatedDocument/parentDocument/id SHALL contain the SOP Instance UID of the original DICOM SR document (CONF:1198-8433)
  11. MAY contain zero or one [0..1] componentOf such that it (CONF:1198-30939)
    1. SHALL contain exactly one [1..1] encompassingEncounter such that it (CONF:1198-30940)
      1. SHALL contain at least one [1..*] id (CONF:1198-30941)
      2. SHALL contain exactly one [1..1] effectiveTime (CONF:1198-30943)
      3. MAY contain zero or one [0..1] responsibleParty such that it (CONF:1198-30945)
        1. SHALL contain exactly one [1..1] assignedEntity such that it (CONF:1198-30946)
          1. SHOULD contain zero or one [0..1] assignedPerson OR contain zero or one [0..1] representedOrganization (CONF:1198-30947)
      4. SHOULD contain zero or one [0..1] encounterParticipant, where its type is Physicianof Record Participant2 (CONF:1198-30948)
      5. id In the case of transformed DICOM SR documents, an appropriate null flavor MAY be used if the id is unavailable (CONF:1198-30942)
  12. SHALL contain [1..1] component such that it (CONF:1198-30696, CONF:1198-30697)
    1. Conforms to Findings Section (templateId: 2.16.840.1.113883.10.20.6.1.2)
  13. SHOULD contain [0..1] component such that it (CONF:1198-30698, CONF:1198-30699)
    1. Conforms to DICOM Object Catalog Section (templateId: 2.16.840.1.113883.10.20.6.1.1)
  14. SHALL contain exactly one [1..1] component such that it (CONF:1198-14907)
    1. SHALL contain exactly one [1..1] structuredBody such that it (CONF:1198-30695)
      1. MAY contain zero or more [0..*] component such that it (CONF:1198-31055)
        1. SHALL contain exactly one [1..1] section such that it (CONF:1198-31056)
          1. SHALL contain exactly one [1..1] code (CONF:1198-31057), which SHOULD be selected from ValueSet DIRSectionTypeCodes 2.16.840.1.113883.11.20.9.59 STATIC (CONF:1198-31207)

            For sections listed in the DIR Section Type Codes table, the code element must contain a LOINC code or DCM code for sections that have no LOINC equivalent.
            Note: The section/code SHOULD be selected from LOINC or DICOM for sections not listed in the DIR Section Type Codes table.

          2. SHOULD contain zero or one [0..1] title (CONF:1198-31058)

            There is no equivalent to section/title in DICOM SR, so for a CDA to SR transformation, the section/code will be transferred and the title element will be dropped.

          3. SHOULD contain zero or one [0..1] text (CONF:1198-31059)
          4. MAY contain zero or more [0..*] subject such that it (CONF:1198-31215)
            1. SHALL contain exactly one [1..1] relatedSubject, where its type is Fetus Subject Context (CONF:1198-31216)
          5. MAY contain zero or more [0..*] author such that it (CONF:1198-31217)
            1. SHALL contain exactly one [1..1] observerContext, where its type is Observer Context (CONF:1198-31218)
          6. MAY contain zero or more [0..*] entry, where its type is Procedure Context (CONF:1198-31213, CONF:1198-31214)

            If the service context of a section is different from the value specified in documentationOf/serviceEvent, then the section SHALL contain one or more entries containing Procedure Context (templateId 2.16.840.1.113883.10.20.6.2.5), which will reset the context for any clinical statements nested within those elements

          7. MAY contain zero or more [0..*] entry, where its type is Text Observation (CONF:1198-31357, CONF:1198-31358)
          8. MAY contain zero or more [0..*] entry, where its type is Code Observations (CONF:1198-31359, CONF:1198-31360)
          9. MAY contain zero or more [0..*] entry, where its type is Quantity Measurement Observation (CONF:1198-31361, CONF:1198-31362)
          10. MAY contain zero or more [0..*] entry, where its type is SOP Instance Observation (CONF:1198-31363, CONF:1198-31364)
          11. If clinical statements are present, the section/text SHALL represent faithfully all such statements and MAY contain additional text (CONF:1198-31060)
          12. All text elements SHALL contain content. Text elements SHALL contain PCDATA or child elements (CONF:1198-31061)
          13. The text elements (and their children) MAY contain Web Access to DICOM Persistent Object (WADO) references to DICOM objects by including a linkHtml element where @href is a valid WADO URL and the text content of linkHtml is the visible text of the hyperlink (CONF:1198-31062)
          14. All sections defined in the DIR Section Type Codes table SHALL be top-level sections (CONF:1198-31211)
          15. SHALL contain at least one text element or one or more component elements (CONF:1198-31212)
  15. The physician requesting the imaging procedure (ClinicalDocument/participant[@typeCode=REF]/associatedEntity), if present, SHOULD also be recorded as an informationRecipient, unless in the local setting another physician (such as the attending physician for an inpatient) is known to be the appropriate recipient of the report (CONF:1198-8412)
  16. When no referring physician is present, as in the case of self-referred screening examinations allowed by law, the intendedRecipient MAY be absent. The intendedRecipient MAY also be the health chart of the patient, in which case the receivedOrganization SHALL be the scoping organization of that chart (CONF:1198-8413)
  17. The DICOM Object Catalog section (templateId 2.16.840.1.113883.10.20.6.1.1), if present, SHALL be the first section in the document Body (CONF:1198-31206)
