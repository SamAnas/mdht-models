Patient Referral Act

[Act: templateId 2.16.840.1.113883.10.20.22.4.140]

This template represents the type of referral (e.g., for dental care, to a specialist, for aging problems) and represents whether the referral is for full care or shared care. It may contain a reference to another act in the document instance representing the clinical reason for the referral (e.g., problem, concern, procedure).

  1. SHALL contain exactly one [1..1] templateId ( CONF:1098-30886, CONF:1098-30887 ) such that it
    1. SHALL contain exactly one [1..1] @root="2.16.840.1.113883.10.20.22.4.140"
  2. SHALL contain exactly one [1..1] @classCode="PCPR" (CodeSystem: 2.16.840.1.113883.5.6 HL7ActClass) (CONF:1098-30884)
  3. SHALL contain exactly one [1..1] @moodCode, which SHALL be selected from ValueSet Patient Referral Act moodCode 2.16.840.1.113883.11.20.9.66 STATIC 2014-09-01 (CONF:1098-30885)
  4. SHALL contain at least one [1..*] id (CONF:1098-30888)

    In the case of a Consultation Note where this referral is being fulfilled by this consultation, this id would be referenced in the inFullfilmentOf/order/id of the Consultation Note.

  5. SHALL contain exactly one [1..1] code, which SHALL be selected from ValueSet Referral Types 2.16.840.1.113883.11.20.9.56 DYNAMIC (CONF:1098-30889)
  6. SHALL contain exactly one [1..1] statusCode (CONF:1098-30892)/@code="active" Active (CodeSystem: 2.16.840.1.113883.5.14 ActStatus) (CONF:1098-31598)
  7. SHALL contain exactly one [1..1] effectiveTime (CONF:1098-30893)

    The effectiveTime represents the time when the future referral is intended to take place.

  8. SHOULD contain zero or one [0..1] priorityCode (CONF:1098-32623)
  9. SHOULD contain [0..*] author such that it (CONF:1098-31612)
    1. Conforms to Author Participation (templateId: 2.16.840.1.113883.10.20.22.4.119)
  10. MAY contain zero or more [0..*] entryRelationship such that it (CONF:1098-31604)

    The following entryRelationship represents whether the referral is for full or shared care.

    1. SHALL contain exactly one [1..1] @typeCode="SUBJ" (CONF:1098-31613)
    2. SHALL contain exactly one [1..1] observation such that it (CONF:1098-31605)
      1. SHALL contain exactly one [1..1] @classCode="OBS" Observation (CodeSystem: 2.16.840.1.113883.5.6 HL7ActClass) (CONF:1098-31606)
      2. SHALL contain exactly one [1..1] @moodCode="RQO" (CodeSystem: 2.16.840.1.113883.5.1001 HL7ActMood) (CONF:1098-31607)
      3. SHALL contain exactly one [1..1] code (CONF:1098-31608)/@code="ASSERTION" assertion (CodeSystem: 2.16.840.1.113883.5.4 HL7ActCode) (CONF:1098-31619, CONF:1098-31620)
      4. SHALL contain exactly one [1..1] statusCode (CONF:1098-31614)/@code="completed" Completed (CodeSystem: 2.16.840.1.113883.5.14 ActStatus) (CONF:1098-31615)
      5. SHOULD contain zero or one [0..1] priorityCode, which SHOULD be selected from ValueSet Act Priority 2.16.840.1.113883.1.11.16866 DYNAMIC (CONF:1098-32443)
      6. SHALL contain exactly one [1..1] value with @xsi:type="CD", which SHOULD be selected from ValueSet Care Model 2.16.840.1.113883.11.20.9.61 STATIC 2014-06-19 (CONF:1098-31611)
  11. MAY contain [0..*] entryRelationship such that it (CONF:1098-31635, CONF:1098-31636, CONF:1098-32634)

    The following entryRelationship represents a reference to another act in the document instance representing the clinical reason for the referral (e.g., problem, concern, procedure).

    1. Contains @typeCode="RSON" RSON
    2. Conforms to Indication2 (templateId: 2.16.840.1.113883.10.20.22.4.19:2014-06-09)
  12. MAY contain zero or more [0..*] participant such that it (CONF:1098-32635)
    1. SHALL contain exactly one [1..1] @typeCode="REFT" (CONF:1098-32638)
    2. SHALL contain exactly one [1..1] participantRole such that it (CONF:1098-32636)
      1. MAY contain zero or one [0..1] code, which SHOULD be selected from ValueSet Healthcare Provider Taxonomy (HIPAA) 2.16.840.1.114222.4.11.1066 DYNAMIC (CONF:1098-32637)
