Policy Activity (V3)

[Act: templateId 2.16.840.1.113883.10.20.22.4.61]

A policy activity represents the policy or program providing the coverage. The person for whom payment is being provided (i.e., the patient) is the covered party. The subscriber of the policy or program is represented as a participant that is the holder of the coverage. The payer is represented as the performer of the policy activity.

  1. SHALL contain exactly one [1..1] templateId ( CONF:1198-8900, CONF:1198-10516, CONF:1198-32595 ) such that it
    1. SHALL contain exactly one [1..1] @root="2.16.840.1.113883.10.20.22.4.61"
    2. SHALL contain exactly one [1..1] @extension="2015-08-01"
  2. SHALL conform to Policy Activity template (templateId: 2.16.840.1.113883.10.20.22.4.61)
  3. SHALL contain exactly one [1..1] code, which SHOULD be selected from ValueSet Health Insurance Type 2.16.840.1.113883.3.88.12.3221.5.2 DYNAMIC (CONF:1198-8903)
  4. SHALL contain exactly one [1..1] statusCode (CONF:1198-8902)/@code="completed" Completed (CodeSystem: 2.16.840.1.113883.5.14 ActStatus) (CONF:1198-19109)
  5. SHALL contain exactly one [1..1] performer such that it (CONF:1198-8906)

    This performer represents the Payer.

    1. SHALL contain exactly one [1..1] @typeCode="PRF" (CONF:8907, R2.1=CONF:1198-8907)
    2. SHALL contain exactly one [1..1] assignedEntity such that it (CONF:1198-8908)
      1. SHALL contain at least one [1..*] id (CONF:8909, R2.1=CONF:1198-8909)
      2. SHOULD contain zero or one [0..1] code (CONF:8914, R2.1=CONF:1198-8914), which SHOULD be selected from ValueSet Financially Responsible Party Type Value Set 2.16.840.1.113883.1.11.10416 DYNAMIC (CONF:15992, R2.1=CONF:1198-15992)
      3. MAY contain zero or one [0..1] addr (CONF:1198-8910)
      4. MAY contain zero or more [0..*] telecom (CONF:1198-8911)
      5. SHOULD contain zero or one [0..1] representedOrganization such that it (CONF:8912, R2.1=CONF:1198-8912)
        1. SHOULD contain zero or one [0..1] name (CONF:1198-8913)
  6. SHOULD contain zero or one [0..1] performer such that it (CONF:1198-8961)

    This performer represents the Guarantor.

    1. SHOULD contain zero or one [0..1] time (CONF:8963, R2.1=CONF:1198-8963)
    2. contain zero or one [0..1] @typeCode="PRF"
    3. SHALL contain exactly one [1..1] assignedEntity such that it (CONF:1198-8962)
      1. SHALL contain exactly one [1..1] code (CONF:1198-8968)/@code="GUAR" Guarantor (CodeSystem: 2.16.840.1.113883.5.110 HL7RoleClass) (CONF:1198-16096, CONF:1198-32165)
      2. SHOULD contain zero or one [0..1] addr (CONF:1198-8964)
      3. SHOULD contain zero or more [0..*] telecom (CONF:1198-8965)
  7. SHALL contain exactly one [1..1] participant such that it (CONF:1198-8916)
    1. SHALL contain exactly one [1..1] @typeCode="COV" (CONF:8917, R2.1=CONF:1198-8917)
    2. SHOULD contain zero or one [0..1] time (CONF:8918, R2.1=CONF:1198-8918)
    3. SHALL contain exactly one [1..1] participantRole such that it (CONF:1198-8921)
      1. SHALL contain exactly one [1..1] code (CONF:8923, R2.1=CONF:1198-8923), which SHOULD be selected from ValueSet Coverage Role Type 2.16.840.1.113883.1.11.18877 DYNAMIC (CONF:16078, R2.1=CONF:1198-16078)
      2. SHOULD contain zero or one [0..1] addr (CONF:8956, R2.1=CONF:1198-8956)
      3. SHALL contain at least one [1..*] id (CONF:1198-8922)

        This id is a unique identifier for the covered party member. Implementers SHOULD use the same GUID for each instance of a member identifier from the same health plan (CONF:1098-8984).

      4. SHOULD contain zero or one [0..1] playingEntity such that it (CONF:1198-8932)
        1. SHALL contain at least one [1..*] name (CONF:1198-8930)
        2. SHALL contain exactly one [1..1] sDTCBirthTime (CONF:1198-31344)

          If the covered party's date of birth is recorded differently in the health plan and in the registration/pharmacy benefit summary, use the date of birth as it is recorded in the health plan.
          The prefix sdtc: SHALL be bound to the namespace "urn:hl7-org:sdtc". The use of the namespace provides a necessary extension to CDA R2 for the use of the birthTime element (CONF:1098-31345).

        3. MAY satisfy: This constraint is being overridden as it has been removed in v2 and should never fire
  8. When the Subscriber is the patient, the participant element describing the subscriber SHALL NOT be present. This information will be recorded instead in the data elements used to record member information (CONF:1198-17139)
