Functional Status Result Observation

[Observation: templateId 2.16.840.1.113883.10.20.22.4.67]

This clinical statement represents details of an evaluation or assessment of a patient's functional status. The evaluation may include assessment of a patient's language, vision, hearing, activities of daily living, behavior, general function, mobility, and self-care status. The statement, if present, will include supporting caregivers, non-medical devices, and the time period for which the evaluation and assessment were performed.

  1. SHALL contain exactly one [1..1] templateId ( CONF:13889, CONF:13890 ) such that it
    1. SHALL contain exactly one [1..1] @root="2.16.840.1.113883.10.20.22.4.67"
  2. SHALL conform to Result Observation template (templateId: 2.16.840.1.113883.10.20.22.4.2)
  3. SHALL contain exactly one [1..1] @classCode="OBS" Observation (CodeSystem: 2.16.840.1.113883.5.6 HL7ActClass) (CONF:13905, R2.0=CONF:1098-13905)
  4. SHALL contain exactly one [1..1] @moodCode="EVN" Event (CodeSystem: 2.16.840.1.113883.5.1001 HL7ActMood) (CONF:13906, R2.0=CONF:1098-13906)
  5. SHALL contain at least one [1..*] id (CONF:13907, R2.0=CONF:1098-13907)
  6. SHALL contain exactly one [1..1] code (CONF:13908), which SHOULD be selected from (CodeSystem: 2.16.840.1.113883.6.1 LOINC)
  7. SHOULD contain zero or one [0..1] text (CONF:13926)
    1. text, if present, SHOULD contain zero or one [0..1] reference/@value (CONF:13927)
  8. SHALL contain exactly one [1..1] statusCode (R2.0=CONF:1098-13929)/@code="completed" Completed (CodeSystem: 2.16.840.1.113883.5.14 ActStatus) (CONF:13929, R2.0=CONF:1098-19101)
  9. SHALL contain exactly one [1..1] effectiveTime (CONF:13930, R2.0=CONF:1098-13930)
  10. SHALL contain exactly one [1..1] value (CONF:13932, R2.0=CONF:1098-13932)
    1. If value/@xsi:type=CD, SHOULD contain a code from SNOMED CT (CodeSystem: 2.16.840.1.113883.6.96) (CONF:14234, R2.0=CONF:1098-14234)
  11. SHOULD contain zero or more [0..*] interpretationCode (CONF:13933)
  12. MAY contain zero or one [0..1] methodCode (CONF:13934)
  13. MAY contain zero or one [0..1] targetSiteCode with @xsi:type="CE" (CONF:13935)
  14. MAY contain [0..1] author such that it (CONF:13936)
    1. Conforms to Author from CDA
  15. MAY contain [0..1] entryRelationship such that it (CONF:13892, CONF:14596, CONF:14218)
    1. Contains @typeCode="REFR" REFR
    2. Conforms to Non Medicinal Supply Activity (templateId: 2.16.840.1.113883.10.20.22.4.50)
  16. MAY contain [0..1] entryRelationship such that it (CONF:13895, R2.0=CONF:1098-13895, CONF:14597, R2.0=CONF:1098-14597, CONF:13897, R2.0=CONF:1098-13897)
    1. Contains @typeCode="REFR" REFR
    2. Conforms to Caregiver Characteristics (templateId: 2.16.840.1.113883.10.20.22.4.72)
  17. MAY contain [0..1] entryRelationship such that it (CONF:14465, R2.0=CONF:1098-14465, CONF:14598, R2.0=CONF:1098-14598, CONF:14466, R2.0=CONF:1098-14466)
    1. Contains @typeCode="COMP" COMP
    2. Conforms to Assessment Scale Observation (templateId: 2.16.840.1.113883.10.20.22.4.69)
  18. SHOULD contain zero or more [0..*] referenceRange such that it (CONF:13937)
    1. SHALL contain exactly one [1..1] observationRange such that it (CONF:13938)
      1. observationRange SHALL NOT contain [0..0] code (CONF:13939)
  19. This reference/@value SHALL begin with a '#' and SHALL point to its corresponding narrative (using the approach defined in CDA Release 2, section 4.3.5.1) (CONF:13928)
