Coverage Activity (V3)

[Act: templateId 2.16.840.1.113883.10.20.22.4.60]

A Coverage Activity groups the policy and authorization acts within a Payers Section to order the payment sources. A Coverage Activity contains one or more Policy Activities, each of which contains zero or more Authorization Activities. The Coverage Activity id is the ID from the patient's insurance card. The sequenceNumber/@value shows the policy order of preference.

  1. SHALL contain exactly one [1..1] templateId ( CONF:1198-8897, CONF:1198-10492, CONF:1198-32596 ) such that it
    1. SHALL contain exactly one [1..1] @root="2.16.840.1.113883.10.20.22.4.60"
    2. SHALL contain exactly one [1..1] @extension="2015-08-01"
  2. SHALL conform to Coverage Activity template (templateId: 2.16.840.1.113883.10.20.22.4.60)
  3. SHALL contain exactly one [1..1] code (CONF:1198-8876)/@code="48768-6" Payment Sources (CodeSystem: 2.16.840.1.113883.6.1 LOINC) (CONF:1198-19160, CONF:1198-32156)
  4. contains [1..*] entryRelationship such that it
    1. Conforms to Policy Activity2 (templateId: 2.16.840.1.113883.10.20.22.4.61:2015-08-01)
  5. SHALL contain at least one [1..*] entryRelationship such that it (CONF:1198-8878)
    1. SHALL contain exactly one [1..1] @typeCode="COMP" (CONF:1198-8879)
    2. MAY contain zero or one [0..1] sequenceNumber (CONF:1198-17174)
    3. SHALL contain exactly one [1..1] act, where its type is Policy Activity (V3) (CONF:1198-15528)
  6. MAY satisfy: This constraint is being overridden as it has been removed in v2 and should never fire
