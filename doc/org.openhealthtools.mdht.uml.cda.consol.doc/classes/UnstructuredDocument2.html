Unstructured Document (V3)

[ClinicalDocument: templateId 2.16.840.1.113883.10.20.22.1.10]

An Unstructured Document (UD) document type can (1) include unstructured content, such as a graphic, directly in a text element with a mediaType attribute, or (2) reference a single document file, such as a word-processing document using a text/reference element. For guidance on how to handle multiple files, on the selection of media types for this IG, and on the identification of external files, see the examples that follow the constraints below. IHE's XDS-SD (Cross-Transaction Specifications and Content Specifications, Scanned Documents Module) profile addresses a similar, more restricted use case, specifically for scanned documents or documents electronically created from existing text sources, and limits content to PDF-A or text. This Unstructured Documents template is applicable not only for scanned documents in non-PDF formats, but also for clinical documents produced through word processing applications, etc. For conformance with both specifications, implementers need to ensure that their documents at a minimum conform with the SHALL constraints from either specification.

Unstructured Document (V3) Body Constraints

  1. SHALL contain exactly one [1..1] templateId ( CONF:1198-7710, CONF:1198-10054, CONF:1198-32522 ) such that it
    1. SHALL contain exactly one [1..1] @root="2.16.840.1.113883.10.20.22.1.10"
    2. SHALL contain exactly one [1..1] @extension="2015-08-01"
  2. SHALL conform to US Realm Header2 template (templateId: 2.16.840.1.113883.10.20.22.1.1:2015-08-01)
  3. SHALL contain at least one [1..*] recordTarget such that it (CONF:1198-31089)
    1. SHALL contain exactly one [1..1] patientRole such that it (CONF:1198-31090)
      1. SHALL contain at least one [1..*] id (CONF:1198-31091)
  4. SHALL contain exactly one [1..1] custodian such that it (CONF:1198-31096)
    1. SHALL contain exactly one [1..1] assignedCustodian such that it (CONF:1198-31097)
      1. SHALL contain exactly one [1..1] representedCustodianOrganization, where its type is CDA Custodian Organization (CONF:1198-31098)
  5. SHALL contain exactly one [1..1] component such that it (CONF:1198-31085)
    1. SHALL contain exactly one [1..1] nonXMLBody such that it (CONF:1198-31086)
      1. SHALL contain exactly one [1..1] text (CONF:1198-31087)
      2. If the text element does not contain a reference element with a value attribute, then it SHALL contain exactly one [1..1] @representation="B64" and exactly one [1..1] @mediaType (CONF:1198-7624)
      3. The value of @mediaType, if present, SHALL be drawn from the value set 2.16.840.1.113883.11.20.7.1 SupportedFileFormats STATIC 20100512 (CONF:1198-7623)
